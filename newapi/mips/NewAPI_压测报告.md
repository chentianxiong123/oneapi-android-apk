## 瓶颈诊断 (vmstat @ 50 并发峰值)

| 指标 | 空闲 | 压测峰值 | 结论 |
|------|-----:|---------:|------|
| CPU 用户态 (us) |  1% |      60% | 🔴 应用处理瓶颈 |
| CPU 内核态 (sy) |  1% |      40% | 🔴 上下文切换密集 |
| CPU 空闲 (id)  | 99% |     4-8% | 🔴 CPU 即将耗尽 |
| IO 等待 (wa)   |  0% |      0%  | ✅ 磁盘非瓶颈 |
| 磁盘 IO (bi/bo) |  0 |        0 | ✅ 无磁盘读写 |
| 上下文切换 (cs) | 635 |  14,263 | ⚠️ 每秒 1.4 万次 |
| 网络流量 | 空闲 |    ~5MB/s | ✅ 带宽充足 |

**结论：瓶颈是 CPU，不是 IO/磁盘/网络。**

Go 二进制稳定可靠，0 崩溃 0 超时。换多核 arm64 设备吞吐可翻倍。
